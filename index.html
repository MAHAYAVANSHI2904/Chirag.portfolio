Project 01
Invoice Automation
AP Pulse

OCR + LLM pipeline that reads any invoice, runs 30+ GST/TDS compliance checks, and generates ready-to-import journal entries — built for Indian NBFC accounts payable teams.

Source Code →
Python Streamlit Groq LLaMA Vision PyMuPDF + OpenCV SQLite / Postgres Google Sheets API Tally XML Export

01 Vision OCR

Multi-page PDF → deskew → denoise → LLaMA vision extraction. Learns vendor invoice layouts after first parse.

02 Compliance Engine

GSTIN checksum, ITC eligibility, RCM detection, 206AB risk, MSME terms, duplicate detection — all in one pass.

03 TDS Mapping

30+ sections mapped via priority cascade. Finance Act 2024 compliant — covers 194T, revised 194H, 194IB.

04 Audit Trail

Every check logged to SQLite + Google Sheets. Compliance score 0–100 with full reasoning per invoice.

05 GSTR-2B Reconciliation

Upload GSTR-2B CSV, flag ITC at risk when vendors haven't filed returns.

06 Journal Entries

Double-entry bookkeeping auto-generated. Export as CSV or Tally-ready XML.

Security & Privacy

No secrets in source.
No data leaves without consent.

Both projects are built and audited with the same baseline: nothing sensitive in the repo, nothing sent anywhere without the user knowing.

🔑 Keys via env vars

API keys and service credentials loaded from environment / secrets — never hardcoded.

🗂️ No data files committed

Databases, vendor records, and exports are gitignored across both repos.

🔒 Client-side by default

Nexus-Fin stores everything in-browser. AP Pulse logs only to your own Sheets/DB.

🧹 Dummy data only

All sample invoices and records in both repos use placeholder values.
